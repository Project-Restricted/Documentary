# Описание проекта «КиноBooru»

## 1. Общие сведения о проекте

**Название проекта:** КиноBooru  

**Тип проекта:**  
Веб-приложение (информационно-развлекательный портал)

**Предметная область:**  
Онлайн-каталоги фильмов, пользовательские обзоры и обсуждения кино

**Назначение проекта:**  
Проект «КиноBooru» предназначен для хранения, просмотра и обсуждения фильмов.  
Платформа объединяет каталог фильмов, систему тегов, пользовательские отзывы и интеграцию внешних видеоплееров.

---

## 2. Актуальность проекта

На сегодняшний день существует множество сайтов о кино, однако большинство из них:
- имеют ограниченные возможности фильтрации;
- ориентированы в первую очередь на рейтинги, а не на обсуждение;
- не позволяют гибко искать фильмы по нестандартным признакам (атмосфера, тематика, настроение).

Проект «КиноBooru» решает эту проблему за счёт:
- расширенной теговой системы;
- возможности обсуждать фильмы в формате отзывов;
- простой и наглядной структуры API, пригодной для дальнейшего расширения.

---

## 3. Цель проекта

**Цель проекта** — разработка MVP веб-сервиса для хранения информации о фильмах, их просмотра и обсуждения пользователями с использованием REST API.

---

## 4. Основные задачи проекта

Для достижения поставленной цели в проекте реализуются следующие задачи:

- разработка серверной части (backend) на Django;
- создание REST API для работы с фильмами, пользователями и отзывами;
- реализация системы регистрации и авторизации пользователей;
- организация хранения информации о фильмах (название, описание, постер, видео);
- добавление возможности оставлять отзывы и ставить лайки;
- реализация ролей пользователей (обычный пользователь, модератор, администратор);
- подготовка проекта к демонстрации и защите как MVP.

---

## 5. Функциональные возможности проекта

### 5.1. Возможности для неавторизованных пользователей
- просмотр списка фильмов;
- просмотр подробной информации о фильме;
- поиск и фильтрация фильмов;
- просмотр отзывов других пользователей.

### 5.2. Возможности для авторизованных пользователей
- регистрация и вход в систему;
- просмотр собственного профиля;
- создание отзывов к фильмам;
- лайк и анлайк отзывов;
- отправка заявки на роль модератора.

### 5.3. Возможности для модераторов
- добавление новых фильмов;
- редактирование информации о фильмах;
- загрузка постеров;
- управление тегами фильмов.

### 5.4. Возможности администратора
- управление пользователями;
- назначение ролей;
- доступ к административной панели Django.

---

## 6. Архитектура проекта

Проект построен по клиент-серверной архитектуре.

### Серверная часть
- **Backend:** Django + Django REST Framework  
- **Авторизация:** JWT (JSON Web Token)  
- **База данных:** PostgreSQL  
- **Хранение медиа:** локальное хранилище (MEDIA_ROOT)

### Клиентская часть
- В рамках MVP используется API (frontend может быть реализован отдельно).

---

## 7. Интеграции и внешние сервисы

В проекте используется интеграция с внешними видеоплатформами:
- Rutube
- VK Видео

Видео воспроизводится через встроенные embed-ссылки, без хранения видеоконтента на сервере проекта.

---

## 8. Ограничения проекта

- Проект реализуется в формате MVP (минимально жизнеспособного продукта).
- Основной упор сделан на backend и API.
- Без сложных механизмов безопасности и масштабирования.
- Интерфейс пользователя может быть упрощённым.

---

## 9. Ожидаемый результат

Результатом выполнения проекта является:
- работающее backend-приложение;
- REST API для фильмов, пользователей и отзывов;
- возможность продемонстрировать основные сценарии работы системы;
- готовность проекта к дальнейшему развитию.

---

## 10. Использование проекта

Проект предназначен для:
- демонстрации навыков backend-разработки;
- учебной защиты лабораторной / курсовой работы;
- дальнейшего расширения функциональности.
