# Распределение ролей в команде проекта «КиноBooru»

Проект выполнялся командой из **5 человек**.  
Роли в проекте **совмещались**, что соответствует методическим рекомендациям.

---

## 1. Кирилл — **Backend-разработчик (ответственный)**, Database Engineer, частично DevOps

### Основной вклад
- Разработка backend-части проекта на **Django + Django REST Framework**
- Проектирование и реализация API:
  - авторизация и регистрация пользователей (JWT)
  - работа с фильмами, отзывами и лайками
- Реализация бизнес-логики и сериализаторов
- Исправление ошибок и отладка API
- Интеграция с базой данных **PostgreSQL**

### Дополнительные роли
- Участие в проектировании моделей БД совместно с Database Engineer
- Участие в деплое backend-части на VPS  
  (Gunicorn, виртуальное окружение, миграции)

---

## 2. Юрий — **Frontend-разработчик (ответственный)**

### Основной вклад
- Разработка клиентской части веб-приложения
- Интеграция фронтенда с backend API
- Реализация:
  - страницы списка фильмов
  - страницы фильма
  - механизма авторизации
- Взаимодействие с backend-разработчиком по контрактам API

### Дополнительно
- Участие в UX-решениях  
  (структура страниц, пользовательские сценарии)
- Разработка дизайна интерфейса с использованием **Figma**

---

## 3. Владислав — **Тимлид (ответственный)**, DevOps (организационно)

### Основной вклад
- Координация команды и контроль прогресса разработки
- Проведение синков и уточнение статуса задач
- Принятие архитектурных решений на уровне проекта
- Контроль готовности проекта к защите

### DevOps-роль (организационная)
- Консультации по деплою backend-части
- Контроль возможности развертывания проекта на сервере
- Участие в обсуждении инфраструктуры проекта

> **Примечание:**  
> Тимлид не обязан непосредственно писать код, его роль — организация, контроль и управление процессом разработки, что полностью соответствует методическим рекомендациям.

---

## 4. Евгений — **Database Engineer (ответственный)**, Backend-разработчик

### Основной вклад
- Проектирование структуры базы данных
- Разработка и обсуждение **ER-диаграммы**
- Определение связей между сущностями:
  - фильмы
  - пользователи
  - отзывы
  - теги
- Совместная работа над Django-моделями
- Участие в составлении системных требований проекта
- Помощь в тестировании и отладке API

### Дополнительно
- Консультации по оптимальной структуре моделей
- Помощь в согласовании моделей с бизнес-требованиями

---

## 5. Алексей — **Project Manager (ответственный)**, Business Analyst, System Analyst

### Основной вклад
- Формирование бизнес-требований к проекту
- Подготовка и оформление:
  - бизнес-требований
  - системных требований
  - описания проекта
- Проработка ER-диаграммы
- Участие в определении функциональных сценариев системы

### Дополнительно
- Разработка и оформление **Use Case (юзкейсов)** проекта
- Декомпозиция бизнес-идеи в пользовательские сценарии

---
