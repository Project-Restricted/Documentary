# Системные требования к проекту «CinemaPortal»

## 1. Общие сведения

### 1.1 Назначение документа
Настоящий документ описывает системные требования к проекту CinemaPortal и определяет **функциональные и нефункциональные требования** к системе, а также её основные ограничения и условия эксплуатации.

### 1.2 Область применения
Система предназначена для использования в качестве веб-приложения, предоставляющего API для работы с фильмами, пользователями и отзывами.

---

## 2. Архитектура системы

### 2.1 Общая архитектура
CinemaPortal реализуется по клиент-серверной архитектуре и включает следующие компоненты:
- Backend (REST API)
- База данных
- Клиентское приложение (веб-интерфейс или внешний клиент)

### 2.2 Тип системы
- Web-приложение
- API-ориентированная система (REST)

---

## 3. Функциональные требования

### 3.1 Требования к аутентификации и пользователям

**FR-01**  
Система должна обеспечивать регистрацию пользователей.

**FR-02**  
Система должна обеспечивать аутентификацию пользователей с использованием логина и пароля.

**FR-03**  
Система должна поддерживать авторизацию на основе JWT-токенов.

**FR-04**  
Система должна предоставлять API для получения информации о текущем пользователе.

**FR-05**  
Система должна поддерживать роли пользователей (пользователь, модератор).

---

### 3.2 Требования к управлению фильмами

**FR-06**  
Система должна предоставлять API для получения списка фильмов.

**FR-07**  
Система должна предоставлять API для получения детальной информации о фильме.

**FR-08**  
Система должна поддерживать фильтрацию и поиск фильмов.

**FR-09**  
Система должна позволять модераторам создавать новые фильмы.

**FR-10**  
Система должна позволять модераторам редактировать существующие фильмы.

---

### 3.3 Требования к отзывам

**FR-11**  
Система должна предоставлять API для создания отзывов к фильмам.

**FR-12**  
Система должна предоставлять API для получения списка отзывов к фильму.

**FR-13**  
Система должна поддерживать пагинацию отзывов.

---

### 3.4 Требования к лайкам

**FR-14**  
Система должна поддерживать возможность постановки и снятия лайков с отзывов.

**FR-15**  
Система должна отображать количество лайков у отзыва.

---

## 4. Нефункциональные требования

### 4.1 Производительность

**NFR-01**  
Система должна обрабатывать стандартные пользовательские запросы с временем отклика не более 2 секунд в демонстрационной среде.

---

### 4.2 Безопасность

**NFR-02**  
Система должна ограничивать доступ к защищённым эндпоинтам только для аутентифицированных пользователей.

**NFR-03**  
Система должна обеспечивать защиту токенов аутентификации.

---

### 4.3 Надёжность

**NFR-04**  
Система должна корректно обрабатывать некорректные запросы и возвращать соответствующие HTTP-коды ошибок.

---

### 4.4 Масштабируемость

**NFR-05**  
Архитектура системы должна позволять дальнейшее расширение функциональности без переработки базовой структуры.

---

### 4.5 Совместимость

**NFR-06**  
API системы должно быть совместимо с современными веб-клиентами и REST-инструментами (браузер, Postman, curl).

---

## 5. Требования к среде выполнения

### 5.1 Серверная часть
- Язык программирования: Python
- Веб-фреймворк: Django, Django REST Framework
- СУБД: PostgreSQL (или SQLite для локальной разработки)

---

### 5.2 Клиентская часть
- Веб-браузер (Google Chrome, Mozilla Firefox или аналогичный)
- Поддержка HTTP/HTTPS

---

## 6. Ограничения

- Проект реализуется в учебных целях.
- Отсутствует необходимость в высокой нагрузке и отказоустойчивости.
- Видеоконтент не хранится на сервере проекта.

---

## 7. Критерии приёмки

Система считается соответствующей требованиям, если:
- все заявленные API эндпоинты доступны и работают;
- пользователи могут регистрироваться и авторизовываться;
- реализована работа с фильмами и отзывами;
- разграничены права доступа пользователей.
